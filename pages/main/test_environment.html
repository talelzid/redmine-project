Architecture et Environnement de Test Redmine

Déploiement de votre Instance Redmine (Docker)

Chaque étudiant déploie sa propre instance locale de Redmine 5.1.x via Docker Compose, adossée à une base de données PostgreSQL.

Prérequis

  • Requis Docker Desktop (Windows / macOS) ou le moteur Docker sous Linux — installé et en cours d'exécution
  • Requis Docker Compose — intégré par défaut dans Docker Desktop
  • Requis Un terminal : Invite de commandes, PowerShell, ou Terminal Linux/macOS

Procédure de déploiement

1

Création de l'espace de travail

Créez un répertoire dédié au projet, puis placez-vous à l'intérieur :

mkdir redmine
cd redmine
2

Configuration du fichier Docker Compose

Dans le dossier redmine, créez un fichier nommé exactement docker-compose.yml avec le contenu suivant : 

version: '3.8'

services:
  db:
    image: postgres:15
    restart: always
    environment:
      POSTGRES_USER: redmine
      POSTGRES_PASSWORD: Hdjsy@1hrts
      POSTGRES_DB: redmine
    volumes:
      - pg_data:/var/lib/postgresql/data

  web:
    image: redmine:latest
    restart: always
    ports:
      - "8082:3000"
    environment:
      REDMINE_DB_POSTGRES: db
      REDMINE_DB_USERNAME: redmine
      REDMINE_DB_PASSWORD: Hdjsy@1hrts
      REDMINE_DB_DATABASE: redmine
    volumes:
      - redmine_data:/usr/src/redmine/files
    depends_on:
      - db

volumes:
  pg_data:
  redmine_data:
Notes :
  • Ports (8082:3000) : Redmine écoute sur le port 3000 dans le conteneur, redirigé vers le port 8082 de votre machine.
  • Volumes (pg_data & redmine_data) : assurent la persistance des données entre les redémarrages.
  • Les deux mots de passe doivent être identiques pour que l'application et la base de données puissent communiquer.
3

Lancement des conteneurs

Depuis le dossier redmine, exécutez :

docker compose up -d
Note : Lors du premier lancement, Redmine peut prendre entre 30 secondes et 1 minute pour s'initialiser (création de la structure de base de données).
4

Accès et première connexion

Ouvrez votre navigateur et accédez à :

http://localhost:8082

Connectez-vous avec les identifiants par défaut :

  • Identifiant (Login) admin
  • Mot de passe admin
Sécurité : Dès la première connexion, Redmine vous redirigera automatiquement vers un formulaire exigeant la modification immédiate du mot de passe administrateur par défaut.
5

Validation de l'état (optionnel)

Pour vérifier le statut de vos conteneurs :

docker compose ps

Pour arrêter proprement l'environnement sans supprimer vos données :

docker compose down

Contingence et Communication

Plan de Contingence (Sauvegarde)

Chaque testeur dispose d'une machine virtuelle locale (VM) contenant une instance de Redmine (v. 5.1.1.stable). Cette VM doit être utilisée comme environnement de secours en cas de panne ou d'indisponibilité prolongée du serveur de test principal.
